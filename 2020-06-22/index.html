2020-06-22 .NET Generic Host

查看英文 .NET Generic Host - The ASP.NET Core templates create a .NET Core Generic Host, HostBuilder.

繁體中文 .NET 泛型主機 - ASP.NET Core 範本會建立 .NET Core 泛型主機HostBuilder。

简体中文 .NET 通用主机 - ASP.NET Core 模板会创建一个 .NET Core 泛型主机 HostBuilder。

.NET Generic Host 用比喻來說，就像幾十層大樓的整個結構體和基礎設施。對於三樓沒有沒電梯的建築而行，是完全不同的概念。 如同這文檔的說明，包括了 DI 和 Logging 等等。截圖故意有左邊的部份，就是以後如果要找這部份的主題，甚至就是這一頁，有個方便的入口。
ASP.NET Core Web Application 基本模版的程序入口是 Program.cs。 
          public class Program
          {
              public static void Main(string[] args)
              {
                  CreateHostBuilder(args).Build().Run();
              }
      
              public static IHostBuilder CreateHostBuilder(string[] args) =>
                  Host.CreateDefaultBuilder(args)
                      .ConfigureWebHostDefaults(webBuilder =>
                      {
                          webBuilder.UseStartup();
                      });
          }
這是現在 C# 主流的寫法，如果是命令行的 Hello World，就是直接在 Main 打印。 而這個 ASP.NET 的 App 就是先建一個 Host, 中文就是主機，不管是 Host 或是主機，肯定無法描述一個很大的建築物的框架和基礎設施。 但是最起碼給了一個合理概略性的命名。
Dependency injection 放到左邊的 [Filter by title] 就可以看到這個主題的頁面。
https://docs.microsoft.com/en-us/aspnet/core/fundamentals/dependency-injection?view=aspnetcore-3.1
DI 是第一個大主題，其涵蓋的範圍和深度可以是幾個小時的密集觀念和實做的課程。用比喻來講，這是幾十層甚至摩天大樓的內部運行機制。以消防設施為例，建構時，要預留業界規格空間，到時候在適當時機安裝上去。講的是規格，不是特定品牌的實際產品。其配套設施的概念也是如此。這也是為什麼大廠，不管是做硬件或是軟件的，都在爭取製定規格。
Logging 也是如此。
https://docs.microsoft.com/en-us/aspnet/core/fundamentals/logging/?view=aspnetcore-3.1
早期，相信有很多歷史原因，有許多第三方的providers。 現在基本上，都已集成有用，除非有很大的特別原因，直接使用 .NET 系統，相對容易維護。
